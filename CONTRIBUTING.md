# 🤝 Contributing

Thank you for your interest in improving this guide.

This repository maintains a structured, stability-oriented operational model for running Arch Linux as a desktop system.

It is not intended to replace the Arch Wiki, nor to become a general-purpose opinion forum.

---

## 🎯 Contribution Philosophy

Contributions are welcome when they improve:

- Technical accuracy
- Alignment with Arch Linux news or policy changes
- Driver lifecycle transitions (e.g., NVIDIA series changes)
- Kernel behavior updates
- Security baseline corrections
- Snapshot or rollback workflow improvements
- Clearer explanation of risks or failure modes

This guide is intentionally conservative and stability-biased.

Changes that introduce:
- Personal preference debates
- Desktop environment advocacy
- Opinion-based tooling swaps
- Scope expansion into servers or enterprise compliance
- Significant stylistic rewrites

may be declined in order to preserve the guide’s focus and structure.

---

## 🧩 Types of Accepted Contributions

### ✔ Corrections
- Incorrect package names
- Deprecated commands
- Broken links
- Outdated driver recommendations

### ✔ Arch Transition Updates
- Major driver series changes
- Kernel packaging changes
- Pacman behavior changes
- systemd defaults shifting
- Repo structure changes

### ✔ Risk Clarifications
- Multi-user implications
- Wayland edge cases
- Snapshot caveats
- Security footnotes

---

## ❌ Not in Scope

This repository does not accept:

- Opinion-based rewrites
- “Use X instead of Y” debates
- Feature creep outside desktop operations
- Server or enterprise hardening expansion
- Distribution comparisons

---

## 📝 How to Contribute

1. Fork the repository.
2. Create a focused branch.
3. Keep changes minimal and clearly scoped.
4. Reference Arch Wiki or Arch News when applicable.
5. Open a pull request describing:
   - What changed
   - Why it changed
   - Whether it reflects an upstream Arch change

---

## 🧠 Maintenance Model

This guide is updated in response to:

- Arch Linux news announcements
- Driver lifecycle transitions
- Rolling-release breakage patterns

Not every suggestion will be merged.

Clarity, stability, and alignment with upstream Arch policy take priority over preference.
